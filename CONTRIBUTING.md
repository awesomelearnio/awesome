## Contribution Guidelines

### 1. **Follow the Learning Path Template**

All new topic submissions must use the [official template](https://github.com/awesomelearnio/learning-path).

This ensures consistency across the entire project.

### 2. **Keep It Minimal, Clear & Curated**

Awesome Learn is not an exhaustive resource dump.

Please follow these principles:

- Fewer, higher-quality recommendations.
- No long lists of links.
- No affiliate links.
- Prefer open-source or free resources when possible.
- Prioritize official documentation.
- Keep explanations simple and direct.

### 3. **Structure Matters**

A learning path must be organized from **beginner â†’ intermediate â†’ advanced**, and include:

- Overview.  
- Prerequisites.  
- Step-by-step plan.  
- Essential resources.  
- Practice suggestions.  
- Next steps.  

Do not skip or rearrange sections unless justified.

### 4. **Writing Style**

Please write in a style that is:

- Clear.
- Neutral.
- Beginner-friendly.
- Free of jargon unless explained.
- Professional but not overly academic.

Avoid overly long paragraphs. Keep explanations concise.

### 5. **Link Requirements**

Links must meet the following criteria:

- Use direct URLs (no trackers, no shortened links).
- No referral or affiliate codes.
- Resource must be high-quality and publicly accessible.
- Use HTTPS whenever possible.

Suggested quality indicators:

- Maintained repositories or documentation
- Published books from reputable publishers.
- Recognized online courses.
- Peer-reviewed or instructor-led materials.

### 6. **Submitting a Pull Request**

Before submitting a PR:

1. Fork the repository.
2. Create a branch for your changes.
3. Ensure Markdown formatting is clean.
4. Run a spell-check and proofread.
5. Ensure all sections of the template are filled.
6. Reference related issues, if applicable.

When ready, open a pull request with:

- A clear summary of your changes.  
- Why the contribution improves Awesome Learn.  
- Any links or sources used for verification.  

We may request edits for clarity, formatting, or quality.

## Maintaining New Learning Paths

If you contribute a new learning path and wish to become a long-term maintainer, let us know in your PR.  

Maintainers help ensure:

- Content stays up-to-date.  
- Links remain valid. 
- Improvements are reviewed.  
- Community contributions stay consistent.
